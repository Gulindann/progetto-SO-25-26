# PandOSsh Phase 1 - Implementation Notes

## 1. PCB Manager (pcb.c)
* **List Management:** The module utilizes the Linux-like list macros provided in `listx.h` (e.g., `list_for_each_entry`) for all queue manipulations, ensuring code consistency and type safety.
* **Safe Initialization:** The `allocPcb` function performs a complete reset of all PCB fields (pointers, processor state, statistics) to `0` or `NULL`. This guarantees that reused PCBs do not carry residual data from previous processes.

## 2. ASL Manager (asl.c)
* **Ordered List Strategy:** The Active Semaphore List (`semd_h`) is maintained **sorted by key** (`s_key`). This design choice satisfies the requirement for ordered insertion and enables early termination during search operations when the iterator exceeds the target key.
* **Internal Helper:** A `static` helper function, `getSemd`, centralizes the search logic. This prevents code duplication across `insertBlocked`, `removeBlocked`, and `headBlocked`.

## 3. General Notes
* **Visibility:** All internal data structures (`pcbFree_h`, `semd_h`) and helper functions are declared as `static` to restrict their scope to the definition file, preventing namespace pollution.

# PandOSsh Phase 2 - Implementation Notes

## 1. Nucleus Modularization
* **Responsibility Segregation:** The Nucleus is decomposed into specialized modules to handle distinct hardware and software events.
* **Unified Entry Point:** A single exception handler in `exceptions.c` coordinates the flow by analyzing the `CAUSE` register and delegating execution to the appropriate sub-handler (Interrupts, Syscalls, or Traps).

## 2. Recursive Process Termination (syscalls.c)
* **Depth-First Progeny Removal:** The `terminateProcessTree` helper function ensures that a process and all its descendants are removed in a single operation.
* **Atomic Cleanup:** The implementation guarantees that if a process is terminated while blocked, its semaphore is adjusted and global counters like `softBlockCount` are updated to maintain system consistency.

## 3. Terminal and Address Management
* **Address-to-Semaphore Mapping:** A centralized logic in `DoIO` and `interrupts.c` maps physical device addresses to kernel semaphores using base-address arithmetic.
* **Terminal Dual-Nature:** Terminals are handled as two independent sub-devices (Receiver and Transmitter) sharing the same base address.
* **Sub-device Identification:** The project distinguishes between these two units by checking the command register offset (specifically `0xC` for transmission), allowing for independent synchronization even though they belong to the same hardware line.

## 4. Standalone Environment Support (utils.c)
* **Compiler-Required `memcpy`:** Since the Nucleus operates without a standard C library, a custom `memcpy` is provided to support implicit structure assignments generated by the compiler (e.g., when saving or restoring `state_t` structures).
* **Precision Timing:** Time accounting is centralized in `utils.c`, leveraging the `STCK` instruction to track CPU bursts accurately across context switches and interrupts.
